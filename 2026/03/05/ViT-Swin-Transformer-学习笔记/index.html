ViT & Swin Transformer 学习笔记

ViT

Transformer 最开始是应用于自然语言处理领域，但是因为其强大的建模能力和全局依赖捕捉能力，人们开始将其应用于视觉领域。ViT 是其中的一个重要的模型，主要是通过将图像切割为固定大小的 Patches 送入 Transformer 进行处理。

我们以 ViT-B/16 为例逐步分析 ViT 的结构。首先是 Embedding 层。经典的 Transformer 的输入是一个 [token_num, token_dim] 的张量，而图像可能是一个 [H, W, C] 的张量。将图像转化为 Patches 的方法不一，这里我们直接用卷积层进行切割。对于 [224, 224, 3] 的图像，使用尺寸为 [16, 16] 的 $768$ 个卷积核按步长 $16$ 进行卷积，转为 [14, 14, 768] 的张量（也可以说是一种下采样操作）。然后将 $14\times 14$ 展平为 $196$（可以视作 $196$ 个 $768$ 维 token），得到了 [196, 768] 的输入矩阵。拼接上 [CLS] token，形状变为 [197, 768]。

随后加入位置编码，形状不变。这里的位置编码和 [CLS] 一样都是可学习的，意在让模型自己学习二维位置编码方式，而不是像经典方法一样手工指定。

经过编码器之后，因为不是生成任务，就不需要解码器了。直接经过一个 LayerNorm 之后，提取出 [CLS] 送入 MLP. 这里实际上是说，用一个 [CLS] 总结整个 Patch 的信息，然后以此做后面的分类等工作。当然这里的工作不能是太精细化的，因为一个 [CLS] 可能承载不了足够多的信息，如果是分割，一般还是需要全部的 Patch token 作为输出。现在一些其他的 ViT 模型也可能不会保留 [CLS] token。

ViT 的意义在于打破了 CNN 的归纳偏置，也就是局部性和平移不变性假设，用一个更单纯的，参数量更大的 Transformer 模型直接学习图像的原始信息，在更大的规模上取得了更好的效果。而且因为很好地支持并行计算，训练效率也很不错。

Swin Transformer

ViT 直接用卷积层将图像切分为若干 Patches 后使用全局注意力进行处理，但是仍存在一些问题：

  1. 固定大小的 Patch 缺少 CNN 那样的从高分辨率到低分辨率的多级视野；
  2. 固定位置的 Patch 可能无法覆盖边界上的物体，遗漏关键信息；
  3. 全局注意力对算力的消耗过大。

为解决这两个问题，Swin Transformer 提出了对应的解决方案：多级下采样、窗口偏移和相对位置编码。

首先，Swin 希望实现多级的下采样，它将编码器分为多个 Stage，每个 Stage 的分辨率不同，且包含若干 Blocks。例如 Stage 1 的 Patch size 为 $4\times 4$，包含两个 Blocks，而 Stage 2 的 Patch size 为 $8\times 8$，两个 Blocks，Stage 3 的 Patch size 为 $16 \times 16$，包含 6 个 Blocks……就像 CNN 越卷积越小，越卷积越深的设计一样。

具体如何实现从 $4\times 4$ 到 $8\times 8$ 的下采样呢？Swin 设计了 Patch Merging 的下采样机制。具体的说，对于 [B, H, W, C] 的图像张量选取奇数行偶数列、偶数行偶数列、奇数行奇数列、偶数行奇数列分别组成 4 个 [B, H/2, W/2, C] 的小 Patch，然后按照深度堆叠起来，得到 [B, H/2, W/2, 4C] 的张量。随后用一个线性层转换为 [B, H/2, W/2, 2C] 的张量。这种做法与卷积相比似乎更好的保留了原始像素，并且也不影响梯度回传。

其次，为了解决边界问题，Swin 设计了 W-MSA 和 SW-MSA 两种相互对称的特殊多头自注意力。每个 Stage 内的 Blocks 数一定是偶数，也就是一个 W-MSA，一个 SW-MSA 对称放置。

W-MSA 即 Windows Multi-head Self Attention，只在一个窗口内做多头自注意力，而不是全局做。这样能够显著降低算力消耗，但是问题是只能看到一个窗口内部的信息，无法跨区域访问。

这显然不是我们想要的，所以我们引入 SW-MSA。在 SW-MSA 中，我们首先对图像张量进行一次循环的窗口偏移，即窗口的分割网格向右下移动一步。通过这样的移动，我们可以看到每个曾经是边缘的位置都被移动到了某一个窗口内部。如图所示。

但是这样会造成问题，边界上出现了一些不规则的小矩形区域。因为 GPU 的并行计算对于不同尺寸的窗口处理起来很不利，我们不得不将其补成原本大小的窗口但是这样就多出来很多窗口造成了算力的消耗。如何解决呢？我们可以每次把最上方的行移动到最下方，把最左侧的列移动到最右侧，然后重新划分窗口这时候注意到每一个小矩形都被恰好放置在某个窗口中了。如下图。

还是有问题。经过移动之后，本来在左侧的区域到了右侧，模型可能会误以为它们是挨着的。我们只能在用某一块小窗口中的位置查询时，Mask 掉同一窗口中属于其它区域的位置，不查询它们。到此，我们就完善了窗口偏移机制，能够在更优的时间复杂度内，既看到局部，又看到周边。当然，这仍旧不是真正的全局视野，Swin 的全局视野更多的是通过前文提到的不断的下采样来实现的。最后，不要忘了将移动的行和列原封不动的移回来。

最后，Swin 对位置编码也提出了改进。相比于绝对位置，Swin 关注查询位置和被查询位置的相对位置坐标，因为两者一定同属一个窗口，假设窗口尺寸为 $M$，则总共的二维坐标取值个数不过 $(2M-1)^2$ 个。通过数学变换，Swin 将二维坐标转化为一维，并且设计了一个可学习的偏置表 $B$。每一种不同的相对位置关系 $i$ 在表中都有一个 $B_i$ 偏置贡献到注意力公式，即 $\text{Attention}(Q, K, V)=softmax(\dfrac{QK^T}{\sqrt{d_k}}+B_i)V$，以此实现位置编码。
